What KINETIC
actually does

Every feature was built for accuracy and usefulness, not to fill a feature checklist.

01
Real-Time Pose Tracking
MediaPipe Pose Landmarker processes every camera frame server-side, detecting 33 body landmarks with visibility scoring. Exponential Moving Average smoothing is applied to all joint angles before rep counting — eliminating the jitter that causes false reps in raw pose estimation output.
02
Automatic Rep Counting
Per-exercise detectors use smoothed joint angles to detect the up and down phases of each movement. Rep state transitions are tracked via stage flags — no button press required. Just perform the movement and KINETIC increments the count when a full range of motion is completed.
03
LLM Form Analysis
Raw pose metrics — knee angles, hip alignment, back angle, swing status — are sent as structured JSON to LLaMA-3.3-70B running on Groq. The model infers what the form issue is from the data and generates a coaching cue. No hardcoded rules. The coaching adapts to what it actually observes.
04
Voice Coaching Pipeline
The LLM output is passed to gTTS and played back through the browser audio API. Coaching fires on major events (workout start, set complete, session end) and on form deviations, with a 5-second cooldown preventing repetitive feedback. Completely hands-free.
05
Progress Analytics
Completed sets are written to SQLite keyed by username and date. The analytics endpoint groups reps and duration by exercise across sessions. Chart.js renders the history as a bar chart with per-exercise filtering — turning the database into a visual progress tracker.
06
WebSocket Architecture
Browser sends JPEG frames over a persistent WebSocket. Ack-based flow control prevents frame queue buildup under server load — the next frame is only sent after the server responds to the previous one. Exponential backoff reconnection handles brief network drops without killing the session.
02

From camera
to coaching

The technical pipeline from your webcam to the coach's voice — in four steps.

01
Configure your session
Choose an exercise and set your target sets and reps in the sidebar. Enter a username — no account, no password. Your session is identified locally and workout history is stored under that username in SQLite.
02
Camera streams frames
Your webcam captures frames at up to 10fps. Each frame is JPEG-encoded in the browser and sent as base64 over a WebSocket to the FastAPI backend. Ack-based flow control ensures the server is never flooded — the browser waits for a response before sending the next frame.
03
MediaPipe analyses pose
The server runs MediaPipe Pose Landmarker on each frame, extracting 33 landmarks. EMA smoothing cleans the angle signal. The exercise-specific detector processes smoothed angles to update the rep count and form flags. The annotated frame is JPEG-encoded and returned to the browser.
04
LLM coaches, voice delivers
Form metrics are serialised as JSON and sent to LLaMA-3.3-70B on Groq, which decides what coaching cue fits the current state. The response is passed to gTTS, and the audio plays in your browser — all within the same WebSocket response cycle.

Common
questions

Nothing to install, no sign-up. KINETIC runs entirely in your browser. Enter a username — which identifies your local workout history — allow camera access, and start training.
No. Frames are processed in real-time on the server and immediately discarded. Only the resulting metrics — rep count, set progress, form flags — are returned to your browser. No video is stored at any point.
Raw joint angles and pose flags are serialised as JSON and sent to LLaMA-3.3-70B running on Groq. The model reads the actual numbers and decides what coaching cue to give — there are no hardcoded if/else rules for form detection. The LLM infers issues from the data the same way a coach would from watching you.
Yes. Pose tracking, rep counting, form metrics, session history, and the progress chart all work without a Groq key. Voice coaching requires the GROQ_API_KEY environment variable set before starting the server.
Raw MediaPipe output jitters frame-to-frame due to detection noise. Without smoothing, a single noisy frame can trigger a false rep count. Exponential Moving Average applies weighted averaging — prioritising recent values while dampening spikes — the same technique used in motion capture pipelines. Alpha 0.35 was chosen for the balance between responsiveness and stability.
Completed sets are written to a local SQLite file (data.db) on the server, grouped by username and date. The /api/analytics endpoint aggregates this into daily totals per exercise, which the frontend renders as a Chart.js bar chart. Deleting data.db resets all history.
